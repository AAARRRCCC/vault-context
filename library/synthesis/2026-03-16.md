## Cross-Project Connections

The strongest cross-cutting signal is **Mayor-Worker hardening via the Symphony pattern**. OpenAI's Symphony SPEC.md, Prosser's Chief of Staff, and the `claude -p` programmatic CLI tweet all describe the same system Brady already runs — but with retry logic, hot-reload, and formalized lifecycle states that Mayor-Worker lacks. This is the highest-leverage improvement across the entire tweet library because it touches the infrastructure layer that every other project depends on.

The second signal Brady hasn't acted on: **token compression**. RTK (transparent CLI proxy that compresses shell output before it hits the context window) would benefit every Worker execution and every relay call through the Foreman bot. It's a one-time install with immediate, compounding returns.

The third: **skill-creator's Improve mode**. Brady has custom skills (process-work-orders, tweet research) that were hand-written and never formally evaluated. Running the official skill-creator's eval/improve loop against them would surface prompt-level improvements without architectural changes.

## WO Sketches

### WO Sketch: Install RTK Token Compression
**Impact:** high  **Effort:** small  **Connects to:** Mayor-Worker, Foreman relay
**Inspired by:** rtk-rust-token-killer

Install RTK globally (`brew install rtk && rtk init --global`). It hooks into `~/.claude/settings.json` and transparently compresses shell output before it enters the context window. Test against a typical WO execution and measure token delta. This is a 15-minute install with permanent savings on every `claude -p` invocation — Worker, relay, and interactive sessions.

### WO Sketch: Mayor-Worker Retry & Lifecycle Formalization
**Impact:** high  **Effort:** medium  **Connects to:** Mayor-Worker dispatch
**Inspired by:** openai-symphony, chief-of-staff

Read Symphony's `SPEC.md` and extract three features to port: (1) exponential backoff retry on Worker failure (currently a WO just errors and sits), (2) terminal-state detection so completed/failed WOs are moved out of the poll path, (3) hot-reload of the Worker's system prompt from a versioned file (currently baked into the command). Scope as a 3-phase plan. This formalizes the dispatch system from "it works" to "it's robust."
